## Description

The FIle organizer Utility is a Python-based automation tool for scannina user-specificed directory, categorizing files based on extensions, and automatically moving said files into subfolders. This project displays practical file-system automation, error handling and modular design. It was developed as part of a project to showcase Python fundamentals and real world problem solutions. 


The Primary goal of this project is to simplify the process of managing cluttered folders. Many people accumulate a large mess of mixed files, like documents, images, audio, and miscellaneous other files. Such a scenario makes it difficult to navigate and manage content. This utility solves that problem by automatically organizing the files into proper categories. It also generates a log file for recording actions taken in a move to provie transparency and traceability.

This project is designed with complexity and space for future evolution. It include multiples functions, robust error handling, user input validation, and file-system operations. It demanded the learning of methods and functions related to handling os level type operations. This also demonstrates defensive programming techniques essential for IT environments where permissions, missing files, and unexpected errors may be common.

## How the Program Works

When the user runs the program, they are prompted to enter a folder path. The program validates the path, scans the directory, identifies each file’s type based on its extension, and moves the file into respective subfolders. Supported categories being:

- **Images** (jpg, jpeg, png, gif)  
- **Documents** (pdf, docx, txt)  
- **Audio** (mp3, wav, aiff, aac)  
- **Other** (any unsupported extension)

A log file (`log.txt`) is updated with each action for errors and successes.

## Key Design Decisions

### **1. Modular Function Design**
Each major task—scanning, categorizing, moving, and logging is handled by a separate function. This ensures readability, and maintainability. It also aligns with best practices in IT scripting and automation.


### **2. Error Handling**
Real world file systems can be unpredictable. FIles can be locked, missing, or inaccessible for a myriad of reason. The program includes handling using try/except blocks for critical operations to ensure crashes are prevented and provide helpful feedback.

### **3. Path Normalization**
User inputs for paths are cleaned using `.strip()` and `.replace("\\", "/")` to avoid common Windows escape‑sequence issues. This ensures the program behaves consistently for users.

### **4. Logging for Transparency**
A log file is essential for auditing and troubleshooting. The log files provides clairty on issues and reduces the need to rerun the program as a user to identify issues. 

### **5. Simplicity and Practicality**
The project focuses on solving a real problem with clear, understandable code. It avoids unnecessary complexity while still demonstrating structured usability and robust usability.
